XSL sheets

These stylesheets can be used to transform your Xmlized objects into a handy HTML form. See the sample output.

  • default.xsl . A skeleton HTML document that imports the following XSL documents.
  • identity.xsl . As the most general XSL template, it copies elements verbatim to the output if no better XSL template is encountered (such as those below).
  • object.xsl . Wraps a generic <h:object> tag in a <span> and gives the <span> a title attribute for browsers that support tooltips. The title includes the object's name, if given (see map.xsl below), and the object's class name.
  • array.xsl . Matches <h:object-array> and <h:object-array> tags. The template creates an unordered-list <ul> with each object in the array being wrapped in its own <li> tag.
  • map.xsl . Matches <h:object> tags whose class is one of Java's Map implementations such as java.util.HashMap and java.util.Properties. A table is constructed with table headers "key" and "value", though these values can be over-ridden by applying the template with the <xsl:with-param> tag. Each key/value pair in the Map is a row in the table. When <xsl:apply-templates> is called on the value object, the key is passed as the object's name which may subsequently appear in tooltips (see object.xsl above).
  • collection.xsl . Matches <h:object> tags whose class is one of Java's Collection implementations such as java.util.ArrayList and java.util.Vector. The template creates an unordered-list <ul> with each object in the collection being wrapped in its own <li> tag. The objects' templates are applied with the Collection's name, if there is one, and the order in which the object appears in the Collection. For instance, the first object's name might be things.to.buy[1].
